1. 🗄️ THE SINGLE SHARED DATABASE & SCHEMA LAYOUT
===============================================================================

database. Data separation is strictly enforced via a 'tenant_id' UUID.

3. 🏎️ INDEXING & FULL-TEXT SEARCH SCALING BLUEPRINT (MILLIONS OF RECORDS)
===============================================================================
To ensure searches take milliseconds, even with millions of candidates:

A. ROW ACCESS ACCELERATION (GIN / B-TREE INDEXING):
   We create index pointers so PostgreSQL skips checking other tenants' data.
   
    [Recruiter searches Candidates]
                  │
                  ▼
         [Postgres Index]  ───(B-Tree Jump)───►  [Tenant Data Block] (1-3ms)
                  │
                  ▼
         (Skips other 99% of database rows!)

B. RESUME SEARCH ACCELERATION (FULL-TEXT INVERTED INDEX):
   Raw resume HTML/Text parsed by Python/Node is converted to a 'tsvector' tokenized 
   index. GIN index speeds up matching words instantly.
   
    Resume Content: "Expert in Spring Boot, AWS, Kubernetes and Microservices..."
                                        │
                                        ▼
    Tokenized Index: 'aws':2 'boot':1 'kubernetes':3 'microservic':4 'spring':5
                                        │
                                        ▼
    GIN Inverted Index Search Match ──► Instant Match inside pg! (5-10ms)

===============================================================================
4. 🤖 AI-POWERED SEMANTIC SEARCH (FUTURE SCALING)
===============================================================================
Using Supabase's `pgvector` extension, we generate AI embeddings for candidates 
to match jobs conceptually instead of raw string matches.

    "Experienced Cloud Architect with focus on high traffic APIs"
                                 │ (AI Embedding)
                                 ▼
                     Vector: [0.12, -0.42, 0.91, ...]
                                 │ (Vector Cosine Distance Match)
                                 ▼
           Matches Candidate resumes with: "AWS DevOps", "Azure API Expert",
           or "Kubernetes Architect" without exact word matches!
